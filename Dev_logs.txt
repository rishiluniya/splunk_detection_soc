Day 1
I am currently using a Linux system(Ubuntu 22.04). As I am aiming to work with Windows logs, I set up 2 VMs. A Kali Linux and a Windows 10 system. Initially I wanted the windows system to monitor the logs using splunk but in the industry generally an admin computer manages all the logs collected from different system, so for our demo our host system will act as the log manager, Windows System will be the victim and Kali will be the attacker. 


Day 2
The first thing I did was understand how splunk looks at logs. I spent a lot of time understanding how indexes work and how the SPL (Splunk Processing Language) works. For the implementation part we started with installing splunk on our host Linux. I am working with the trial version of the Splunk Enterprise. Installing was pretty smooth. It works as a server on your system and it runs on a specific port and you can access the GUI using a web browser. I ran it on the default port of 8000. Then I set up a receiving port (9997) so that Splunk receives data. 
Then on the Windows 10 system I installed the Splunk Universal Forwarder and I found out about conf files that need to be established. The outputs.conf decided where to send the data and the inputs.conf  decide what logs to monitor. Learning about the conf files was important as they have very specific syntax and I got a default file from the internet and modified it. For collecting logs I was using sysmon and I found out that you need a configure file that tells it what needs to be monitored like: Process creation, Network connections, File creation, etc. The next step was to turn on powershell logging using gpedit. There I rea;ised that my Winodws home edition does not have the provision to access gpedit.msc. I treid to look at alternatives but those were either too complicated or not reliable. So I deleted the entire Vm and upgraded to and redid all the steps. After doing all this I restarted evertything and tested it out.
Here I ran into another problem because SPLUNK wasn't showing any logs in its database. After debugging I realized that I had forgotten to create an index where all the data should be directed(silliest mistake!). After configuration, finally all the data from windows was being sent and I could see the Event codes and I tested some queries which were giving correct outputs.
